Ciclo de Fluxo de Trabalho do Git e GitHub

Breve descricao

Clone

O comando git clone é usado para criar uma cópia idêntica de um repositório Git remoto na sua máquina local.

Exemplo:

git clone https://github.com/nome-do-usuario/nome-do-repositorio.git

Branch

o comando cria uma nova ramificação (branch) usando git branch ou git checkout -b. As ramificações são usadas para desenvolver funcionalidades isoladamente sem interferir no código principal.

Exemplo:

git branch < nome da branch>

Commit


As modificações feitas nos arquivos são registradas através de commits usando git commit. Cada commit tem uma mensagem descritiva que explica as alterações realizadas.

Exemplo:

git commit -m "commit message"

Merge


Após revisões e aprovações, o pull request é mesclado (git merge) na ramificação principal. Isso incorpora as alterações ao código principal do projeto.

Exemplo:

git merge nova-feature

Pull Request


Se o desenvolvedor criou uma nova funcionalidade ou corrigiu um bug em uma ramificação separada, ele pode abrir um pull request no GitHub. Isso permite que outros revisem as alterações antes de serem mescladas à ramificação principal.

Exemplo:

Clicke em “OPEN A PULL REQUEST” e adicione um comentario sobre as modificações feitas.
